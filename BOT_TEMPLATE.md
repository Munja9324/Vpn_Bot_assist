# Telegram VPN admin template

## Bot

Username бота: `@vpn_kbr_bot`

## Input

`ID_USER` - ID пользователя из команды `help ID_USER`.

Пример:

```text
help 123456789
```

## Flow

1. Открыть `@vpn_kbr_bot`.
2. Отправить `/admin`.
3. Нажать `Пользователи`.
4. Нажать `Найти пользователя`.
5. Вписать `ID_USER`.
6. Нажать `Подписки пользователя`.
7. Из текста `Управление VPN пользователя №...` определить номер пользователя.
8. Из кнопок вида `[932] Финляндия` собрать все номера подписок.

## Output

`ID_PODP` - айди подписок пользователя через запятую.

Если подписок нет:

```text
1. Username бота: @vpn_kbr_bot
2. ID пользователей: 123
3. Айди подписки пользователей: подписок нет
```

Если подписки есть:

```text
1. Username бота: @vpn_kbr_bot
2. ID пользователей: 123
3. Айди подписки пользователей: 932, 933
```

## Mail command

Если отправить в личный чат:

```text
mail 123456789 Текст сообщения
```

Сценарий:

1. Открыть `@vpn_kbr_bot`.
2. Отправить `/admin`.
3. Нажать `Пользователи`.
4. Нажать `Найти пользователя`.
5. Вписать ID пользователя.
6. Нажать `Написать пользователю`.
7. Вписать текст сообщения.
8. Нажать `Далее`.

## Unsupported command

## Info command

```text
info 123456789
```

Команда собирает тексты всех подписок пользователя одним ответом.

## Scan command

```text
scan
scan pause
scan reset
scan results
```

Команда проходит по страницам пользователей в одном подробном режиме, собирает подписки и сохраняет отчеты в `reports/`.
Пауза сохраняет прогресс, следующий `scan` продолжит по несобранным ID.
Сброс очищает сохраненный прогресс, готовые отчеты остаются.

Если сообщение не начинается с поддерживаемой команды `help` или `mail`, ответ:

```text
Такой команды нет в списке поддерживаемых.
```
